# ATLAS BANK - ENTERPRISE TEST SUITE

This document outlines the testing architecture and execution procedures for the Atlas Bank Fullstack Application.

## 🧪 Testing Philosophy
As a banking application, **Security**, **Data Integrity**, and **Atomic Transactions** are non-negotiable. This test suite is designed to ensure 100% compliance with financial regulations and zero-tolerance for balance inconsistencies.

## 📁 Directory Structure
- `tests/unit/`: Tests for individual functions and classes (helpers, audit logging).
- `tests/integration/`: Tests for API endpoints and their interaction with the database.
- `tests/financial/`: Critical tests for double-spending, concurrency, and financial precision.
- `tests/security/`: Penetration testing simulations (RBAC bypass, SQLi, XSS, Token misuse).
- `tests/regression/`: Tests for previously fixed bugs to prevent recurrence.
- `tests/consistency/`: Data transformation and precision loss verification.

## 🚀 Execution Instructions

### Prerequisites
- PHP 8.1+
- PHPUnit 10+ (Recommended)
- Local MySQL instance with `atlas_bank` database.

### Running Tests with PHPUnit
To run the entire suite:
```bash
phpunit tests
```

To run specific categories:
```bash
phpunit tests/unit
phpunit tests/financial
```

### Manual Verification
Each test file contains a structured header:
1. **Test Name**
2. **What it validates**
3. **Setup**
4. **Steps**
5. **Expected Result**
6. **Failure condition**

You can follow these steps manually if a automated runner is not available in your environment.

## 🔴 Critical Paths (High Priority)
1. **Transaction Atomicity**: Balance update MUST succeed if and only if the transaction record is created.
2. **Concurrency**: Multiple simultaneous withdrawals must never lead to a negative balance.
3. **Branch Isolation**: Staff must never see data outside their assigned branch unless authorized.
4. **Fee Calculations**: Precision must be maintained to 2 decimal places with proper rounding (Half-Up).

## 🛡️ Security Hardening
The suite includes tests for:
- JWT Token expiration and revocation.
- Role-Based Access Control (RBAC) enforcement.
- Input sanitization (SQLi and XSS protection).
- CSRF Token rotation and validation.
